# Minishell — Простейший командный интерпретатор Unix

Проект представляет собой собственный легковесный Unix-шелл (интерпретатор командной строки), написанный на языке Go. Программа реализует базовый REPL-цикл, поддерживает работу со встроенными и внешними командами, управление процессами, конвейеры (пайплайны) и условные операторы выполнения.

## Функциональные возможности

### 1. Встроенные команды (Builtins)
Выполняются внутри основного процесса командного интерпретатора:
* `cd <path>` — смена текущей рабочей директории (при отсутствии аргумента переводит в домашнюю директорию).
* `pwd` — вывод абсолютного пути текущей директории.
* `echo <args>` — вывод переданных аргументов в стандартный поток вывода.
* `ps` — вывод списка запущенных процессов. Реализован кроссплатформенный подход: честный парсинг виртуальной файловой системы `/proc` в Linux и вызов системной утилиты на macOS.
* `kill <pid>` — отправка сигнала завершения процесса `SIGKILL` по его идентификатору.
* `exit` — корректное завершение работы самого шелла.

### 2. Внешние команды
* Запуск любых внешних программ и утилит, доступных в системе (`ls`, `grep`, `top`, `cat` и др.), через системные вызовы `fork/exec` (с использованием пакета `os/exec`).

### 3. Конвейеры (Pipelines)
* Возможность объединения нескольких команд в цепочки с помощью символа `|`. Вывод (`Stdout`) предыдущей команды автоматически перенаправляется на ввод (`Stdin`) следующей.
* *Пример:* `ps | grep go | wc -l`

### 4. Обработка сигналов и завершения
* **`Ctrl+D` (EOF):** шелл корректно завершает свою работу и освобождает ресурсы.
* **`Ctrl+C` (SIGINT):** прерывает выполнение текущей запущенной внешней команды, но не закрывает сам интерактивный шелл.

### 5. Дополнительный функционал
* **Условное выполнение:** поддержка операторов `&&` (выполнить, если предыдущая команда завершилась успешно) и `||` (выполнить, если предыдущая завершилась с ошибкой).
* **Переменные окружения:** автоматическая подстановка значений переменных вида `$VAR`.
* **Редиректы:** перенаправление потоков данных `>` (вывод в файл) и `<` (чтение из файла).

## Архитектура проекта

Программа построена по классическому принципу REPL (Read-Eval-Print Loop):

1. **Read (Чтение):** Потокобезопасное чтение строк из `os.Stdin` с помощью `bufio.NewScanner`.
2. **Eval (Парсинг и Выполнение):** * Строка токенизируется и очищается от лишних пробелов.
    * Контекст проверяется на наличие спецсимволов (`|`, `&&`, `||`, `>`, `<`).
    * Настраиваются каналы взаимодействия процессов (`io.Pipe`) и файловые дескрипторы для редиректов.
    * Команды отправляются на выполнение в соответствии со своим типом (встроенная или внешняя).
3. **Print (Вывод):** Результаты выполнения выводятся в системные потоки `os.Stdout`/`os.Stderr`.
4. **Loop (Цикл):** Буферизированный вывод сбрасывается (`Sync`), и шелл возвращается в состояние ожидания ввода.

## Требования

* Установленный Go (версии 1.16 или выше).
* Unix-подобная операционная система (Linux для полной работы `/proc`, macOS/BSD — с ограничениями).
