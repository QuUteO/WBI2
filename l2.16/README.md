# Mini-Wget — Утилита зеркалирования веб-сайтов

Проект представляет собой консольную утилиту на языке Go для загрузки веб-страниц вместе со всем вложенным контентом (ресурсы, ссылки), аналогичную системной команде `wget -m`. Программа рекурсивно скачивает HTML-страницы и медиа-ресурсы, формируя полностью автономный локальный каталог для офлайн-просмотра сайтов.

## Функциональные возможности

### 1. Рекурсивный обход и парсинг
* **Ограничение глубины:** Скачивание страниц вглубь до заданного уровня ограничения (`-depth`).
* **Изоляция домена:** Защита от «ухода» краулера на сторонние ресурсы (скачиваются файлы только в рамках стартового домена).
* **Поддержка медиа-ресурсов:** Автоматическое обнаружение и скачивание картинок (`<img>`), стилей (`<link href>`), скриптов (`<script src>`), аудио и видео материалов.

### 2. Локализация контента (Локальные ссылки)
* **Мутация HTML-дерева:** Автоматический парсинг HTML-документов и перезапись ссылок на корректные относительные локальные пути.
* **Парсинг CSS:** Извлечение и скачивание ресурсов, скрытых внутри CSS-файлов через конструкции `@import` и `url()`, с их последующей локализацией.

### 3. Умная дедупликация и защита от зацикливания
* **Потокобезопасный кэш:** Использование внутренней карты посещенных URL под защитой `sync.Mutex` для исключения повторного скачивания одного и того же ресурса.
* **Очистка URL:** Вырезание фрагментов (хэшей `#`) для предотвращения дублирования запросов к одной и той же странице.
* **Хеширование Query-параметров:** Применение алгоритма SHA-1 к GET-параметрам (строке запроса) для корректного разделения и сохранения страниц с динамическим контентом.

### 4. Сетевой слой и обработка ошибок
* **Отказоустойчивость:** Настройка жестких таймаутов на сетевые запросы, валидация HTTP-статус кодов ответа сервера.
* **Маскировка:** Подстановка кастомного заголовка `User-Agent` для предотвращения блокировок со стороны систем защиты сайтов.

## Архитектура проекта

Программа построена по модульному принципу и разделена на три ключевых слоя:

1. **Сетевой слой (`package downloader`):**
    * Отвечает за низкоуровневые HTTP-запросы.
    * Управляет конфигурацией HTTP-клиента, таймаутами и заголовками запросов.
2. **Контентный слой (`package parser`):**
    * Превращает сырые байты данных в структурированное дерево HTML-узлов с помощью `golang.org/x/net/html`.
    * Извлекает ссылки, фильтрует домены по белому списку, рассчитывает локальные пути (`filepath.Rel`) и переписывает атрибуты тегов.
3. **Слой оркестрации (`package crawler`):**
    * Управляет циклом рекурсивного обхода, координирует взаимодействие загрузчика и парсера, отвечает за создание структуры папок на диске и потокобезопасный учет состояния.

## Требования

* Установленный Go (версии 1.18 или выше).
* Внешняя зависимость `golang.org/x/net/html` для парсинга DOM-дерева.
